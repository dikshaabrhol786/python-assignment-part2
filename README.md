# Restaurant Order Management System

## 📌 Project Overview

This project is a Python-based Restaurant Order Management System built using core data structures like **lists, dictionaries, and nested structures**.

It simulates real-world restaurant operations including:

* Menu exploration
* Cart management
* Inventory tracking
* Sales analysis

---

## 🎯 Objectives

* Practice Python fundamentals using real-world scenarios
* Work with nested data structures
* Implement business logic like order processing and inventory tracking
* Perform basic data analysis on sales logs

---

## 🧩 Features

### 1️⃣ Menu Exploration

* Displays menu grouped by category (Starters, Mains, Desserts)
* Calculates:

  * Total items
  * Available items
  * Most expensive item
  * Items under ₹150

---

### 2️⃣ Cart Operations

* Add items to cart with quantity
* Prevent duplicate entries (updates quantity instead)
* Validate:

  * Item existence
  * Availability
* Remove items from cart
* Update item quantities
* Generates final order summary with:

  * Subtotal
  * GST (5%)
  * Total payable amount

---

### 3️⃣ Inventory Management

* Uses deep copy to preserve original inventory
* Updates stock after order placement
* Handles insufficient stock cases
* Generates reorder alerts when stock is low

---

### 4️⃣ Sales Analysis

* Calculates daily revenue
* Identifies best-selling day
* Finds most ordered item
* Updates sales log dynamically
* Displays all orders using enumeration

---

## 🛠️ Tech Stack

* Python (Core concepts only)
* No external libraries used (except `copy` for deep copy)

---

## 📂 Data Structures Used

* Dictionaries (menu, inventory, sales_log)

* Nested combinations for real-world simulation
